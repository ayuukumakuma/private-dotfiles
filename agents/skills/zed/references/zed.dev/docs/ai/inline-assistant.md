# [Inline Assistant](https://zed.dev/docs/ai/inline-assistant.html#inline-assistant)

## [Usage Overview](https://zed.dev/docs/ai/inline-assistant.html#usage-overview)

Use `ctrl-enter|ctrl-enter` to open the Inline Assistant in editors, text threads, the rules library, channel notes, and the terminal panel.

The Inline Assistant sends your current selection (or line) to a language model and replaces it with the response.

## [Getting Started](https://zed.dev/docs/ai/inline-assistant.html#getting-started)

If you're using the Inline Assistant for the first time, you need to have at least one LLM provider or external agent configured.
You can do that by:

1. [subscribing to our Pro plan](https://zed.dev/pricing), so you have access to our hosted models
2. [using your own API keys](https://zed.dev/docs/ai/llm-providers.html#use-your-own-keys), either from model providers like Anthropic or model gateways like OpenRouter.

If you have already set up an LLM provider to interact with [the Agent Panel](https://zed.dev/docs/ai/agent-panel.html#getting-started), then that will also work for the Inline Assistant.

> Unlike the Agent Panel, though, the only exception at the moment is [external agents](https://zed.dev/docs/ai/external-agents.html).
> They currently can't be used for generating changes with the Inline Assistant.

## [Adding Context](https://zed.dev/docs/ai/inline-assistant.html#adding-context)

You can add context in the Inline Assistant the same way you can in [the Agent Panel](https://zed.dev/docs/ai/agent-panel.html#adding-context):

* @-mention files, directories, past threads, rules, and symbols
* paste images that are copied on your clipboard

You can also create a thread in the Agent Panel, then reference it with `@thread` in the Inline Assistant. This lets you refine a specific change from a larger thread without re-explaining context.

## [Parallel Generations](https://zed.dev/docs/ai/inline-assistant.html#parallel-generations)

The Inline Assistant can generate multiple changes at once:

### [Multiple Cursors](https://zed.dev/docs/ai/inline-assistant.html#multiple-cursors)

With multiple cursors, pressing `ctrl-enter|ctrl-enter` sends the same prompt to each cursor position, generating changes at all locations simultaneously.

This works well with excerpts in [multibuffers](https://zed.dev/docs/multibuffers.html).

### [Multiple Models](https://zed.dev/docs/ai/inline-assistant.html#multiple-models)

You can use the Inline Assistant to send the same prompt to multiple models at once.

Here's how you can customize your settings file ([how to edit](https://zed.dev/docs/configuring-zed.html#settings-files)) to add this functionality:

```
{
  "agent": {
    "default_model": {
      "provider": "zed.dev",
      "model": "claude-sonnet-4-5"
    },
    "inline_alternatives": [
      {
        "provider": "zed.dev",
        "model": "gpt-4-mini"
      }
    ]
  }
}
```

When multiple models are configured, you'll see in the Inline Assistant UI buttons that allow you to cycle between outputs generated by each model.

The models you specify here are always used in *addition* to your [default model](https://zed.dev/docs/ai/inline-assistant.html#default-model).

For example, the following configuration will generate three outputs for every assist.
One with Claude Sonnet 4.5 (the default model), another with GPT-5-mini, and another one with Gemini 3 Flash.

```
{
  "agent": {
    "default_model": {
      "provider": "zed.dev",
      "model": "claude-sonnet-4-5"
    },
    "inline_alternatives": [
      {
        "provider": "zed.dev",
        "model": "gpt-4-mini"
      },
      {
        "provider": "zed.dev",
        "model": "gemini-3-flash"
      }
    ]
  }
}
```

## [Inline Assistant vs. Edit Prediction](https://zed.dev/docs/ai/inline-assistant.html#inline-assistant-vs-edit-prediction)

Both features generate inline code, but they work differently:

* **Inline Assistant**: You write a prompt and select what to transform. You control the context.
* **[Edit Prediction](https://zed.dev/docs/ai/edit-prediction.html)**: Zed automatically suggests edits based on your recent changes, visited files, and cursor position. No prompting required.

The key difference: Inline Assistant is explicit and prompt-driven; Edit Prediction is automatic and context-inferred.

## [Prefilling Prompts](https://zed.dev/docs/ai/inline-assistant.html#prefilling-prompts)

To create a custom keybinding that prefills a prompt, you can add the following format in your keymap:

```
[
  {
    "context": "Editor && mode == full",
    "bindings": {
      "ctrl-shift-enter": [
        "assistant::InlineAssist",
        { "prompt": "Build a snake game" }
      ]
    }
  }
]
```
