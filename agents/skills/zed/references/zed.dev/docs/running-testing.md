# [Running & Testing](https://zed.dev/docs/running-testing.html#running--testing)

Use this section to run and test code in Zed, then debug issues without
leaving the editor.

## [What's here](https://zed.dev/docs/running-testing.html#whats-here)

* **[Terminal](https://zed.dev/docs/terminal.html)**: Zed's built-in terminal emulator. Open multiple terminals, customize your shell, and integrate with the editor. Tasks and commands run here.
* **[Tasks](https://zed.dev/docs/tasks.html)**: Define and run shell commands with access to editor context like the current file, selection, or symbol. Use tasks to build, lint, run scripts, or execute any repeatable workflow.
* **[Debugger](https://zed.dev/docs/debugger.html)**: Set breakpoints, step through code, and inspect variables using Zed's built-in debugger. Works with C, C++, Go, JavaScript, Python, Rust, TypeScript, and more through the Debug Adapter Protocol.
* **[REPL](https://zed.dev/docs/repl.html)**: Run code interactively using Jupyter kernels. Execute selections or cells and see results inline—useful for Python, TypeScript (Deno), R, Julia, and other supported languages.

## [Quick start](https://zed.dev/docs/running-testing.html#quick-start)

**Open a terminal**: Press `` Ctrl+` `` to toggle the terminal panel, or `Ctrl+~` to open a new terminal.

**Run a command**: Press `Cmd+Shift+R` (macOS) or `Ctrl+Shift+R` (Linux/Windows) to open the task picker, then type any shell command.

**Start debugging**: Press `Cmd+Shift+D` (macOS) or `Ctrl+Shift+D` (Linux/Windows) to open the debug panel and select a configuration.

**Run code interactively**: In a Python or TypeScript file, select some code and press `Ctrl+Shift+Enter` to execute it in a REPL session.
